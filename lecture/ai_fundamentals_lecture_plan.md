# 원론강의 — AI 기본기 커리큘럼과 진행 대본 (초안 v3.1)

> 회계사 대상 **총 2시간 무료 강의**의 **앞 1시간(원론)** 기획서입니다. 뒤 1시간은
> [`pilot_lecture_plan.md`](./pilot_lecture_plan.md)의 실무 시연(완성된 감사 시스템 실연)으로 이어집니다.
>
> **이 1시간이 노리는 것:** AI를 "한 번 써봤다"를 넘어, **원리를 한 번 제대로 짚어** 신뢰를 세우고,
> **강사가 Claude Code를 직접 라이브로 써 보여** "쓰는 단계 → 맡기는 단계(AX)"를 체감시키는 것. 그리고 관심 있는 분들이
> **유료 과정(내 회계실에 전체 세팅하는 단계)**으로 넘어가도록 동기를 만드는 것.
>
> 청중은 **웹 ChatGPT·Claude는 써봤지만 코딩은 안 해본 현직 회계사**입니다. 화면·결과 위주로 가고,
> 코드는 보여주지 않습니다. (작성 기준일: 2026-06-19)
>
> ---

## 오늘의 핵심 명제

> **AI를 "쓰는" 단계에서 "맡기는(AX)" 단계로 넘어가려면, AI가 무엇을 잘하고 무엇을 못하는지를 먼저 알아야 한다.
> AI는 언어는 잘하지만, *말로만* 숫자와 사실을 꺼내면 틀린다 — 그래서 숫자는 계산기(코드)에, 사실은 검색에 맡기고,
> 최종 검산·판단·서명은 회계사가 한다.**

이 명제가 1시간의 중심축입니다. 전반부 개념(원리·AX·신뢰)과 후반부 실사용(Claude Code 라이브)이 모두 이 한 줄로 수렴하고,
뒤 1시간 실연의 **신뢰 발판**이 됩니다. *"말로만 꺼내면 틀리니까 도구에 맡긴다"*는 말은
뒤 1시간의 "데이터를 정규화해 코드로 결정적 분석을 돌린다"와 **모순 없이 정렬**됩니다.
"자동화가 사람을 대체한다"가 아니라 **"증강(Augment, not replace)"**이라는 회계사 프레임과 일치합니다.

# 전반부. 커리큘럼 — PPT 블록별 설계 (00:00–30:00)

> 각 블록을 *목표 / 청중에게 전할 한마디 / 무엇을 보여줄지* 순으로 정리했습니다. 전반부는 **개념만**, 환각은 캡처로(라이브 재현은 후반부 비트①).

## 오프닝 — "오늘은 원리 30분, 그리고 직접 쓰는 30분입니다" *(4분)*

- **목표:** 강의가 **2구조(전반부 PPT / 후반부 Claude Code 라이브)**임을 명시 예고하고, **학습목표 한 줄**을 박고, 뒤 1시간 실연을 예고합니다.
- **학습목표(명시·v3.0 교체):** "오늘 1시간이 끝나면 여러분은, 업무지시서 한 장(**CLAUDE.md**)을 직접 써서 AI가 내 기준·내 말투로 답하게 만들어 볼 수 있습니다." → 후반부 비트②의 정당성이 되고, 클로징에서 회수합니다.
- **전할 한마디:** "이미 AI는 쓰고 계시죠 — 업계 설문(FindSkill.ai, 공식 통계는 아님) 기준 2026년 회계사의 **절반 가까이(약 46%)**가 매일 씁니다 — 정확한 수치보다 '이미 대세'라는 방향이 중요해서 인용합니다. *'쓴다'와 '제대로 쓴다'*는 다릅니다. 오늘 1시간은 두 부분입니다 — **앞 30분은 왜 틀리는지, 어떻게 쓰는지 개념을 잡고, 뒤 30분은 제가 직접 Claude Code라는 도구를 써서 회계 업무에 어떻게 꽂는지 실제로 보여드립니다.** 그리고 뒤이어 1시간은 그 도구로 만든 완성된 회계 시스템을 돌려보여드립니다."
- **보여줄 것:** 오늘의 지도 한 장(**전반부/후반부 2구조 명시**) + **학습목표 한 줄** + 통계 한 줄(46%, 출처 표기) + **뒤 1시간 결과 1장면**(티저, "이건 끝에 보여드릴게요"로만).

## 프롤로그 — 큰 그림: "전문용어, 이미 이 한 문장에 있습니다" *(4분, 명제의 범례)*

- **목표:** 회계사가 모를 전문용어(LLM·RAG·에이전트·하네스)를 **새로 가르치지 않고**, "오늘의 명제 한 문장 안에 이미 다 들어있다"는 사실을 한 장으로 폭로합니다. 이후 전·후반부에서 용어가 나올 때마다 청중이 "아, 벌써 본 거지"가 되게 합니다(**복습** 구조).
- **전할 한마디:** "오늘 들으실 전문용어 — LLM, 환각, RAG, 에이전트, 하네스 — 처음 들으시는 것 같지만, **방금 보여드린 명제 한 문장 안에 전부 들어있습니다.** 어디에 있는지 한 번 짚어드리겠습니다."
- **보여줄 것:**
  1. **발전사 1줄(슬라이드)**: 규칙(엑셀 매크로·VLOOKUP) → 통계·머신러닝(데이터에서 패턴) → **LLM(언어, ChatGPT/Claude — "지금")** → **에이전트/AX(LLM에 손발·도구를 단 응용 단계 — "다음")**. *"엑셀이 1단계, 데이터에서 패턴 찾는 통계·머신러닝이 2단계, 지금 ChatGPT/Claude로 글 쓰는 게 3단계 LLM, 그 LLM에 도구를 달아 일을 끝까지 맡기는 게 에이전트 — 새 두뇌가 아니라 3단계 위의 응용 단계. 지금 3단계 한가운데서 이 응용으로 넘어가는 중입니다."*
  2. **"한 장 큰 그림"(개념 지도)** — 중앙에 **LLM 두뇌**, 주변에 5용어 배치, 상단에 명제 문장을 띄움:
     ```
             [ 사실 검색(조문·예규 코퍼스) ] ← RAG
                        ↘
       [코드/계산기] ← [ LLM 두뇌 ] → [ 환각(말로만 틀림) ]
       "숫자는 계산기에"    "언어는 잘하는"     "말로만 꺼내면 틀린다"
                       ↓ [ 하네스: 실행 틀 + 고삐(CLAUDE.md·Skill) ]
                     [ 에이전트/AX — "일을 끝까지 맡기는 단계" ]
     ```
     - **LLM** = "언어는 잘하는" 두뇌(ChatGPT·Claude)
     - **환각** = "말로만 숫자·사실을 꺼내면 틀린다"
     - **RAG** = "사실은 검색에 맡긴다" — 내가 통제하는 문서 더미(조문·예규·판례 같은 *텍스트 사실*)에서 필요 부분을 찾아 AI에 주입. (숫자는 RAG가 아니라 아래 계산기에.)
     - **코드/계산기** = "숫자는 계산기에" — 계좌잔액·누적과세구간·차대합계 같은 숫자는 여기서(환각이 안 터지는 자리 — 뒤 1시간 원천세 실연의 뼈대).
     - **하네스** = 에이전트가 돌아가는 실행 틀(도구·가드레일 포함). 그 위에 올리는 업무지시서(CLAUDE.md)·절차 매뉴얼(Skill) 2개가 "내 기준대로 일하게" 잡는 고삐 역할. **→ "이 두 고삐를 뒤 30분에 직접 만들어 보여드리겠습니다."**
     - **에이전트/AX** = "맡기는 단계" — 일을 끝까지 수행 *(실연은 뒤 30분 + 뒤 1시간에)*
  3. **범례 한 줄(핵심)**: **"이 한 문장 안에 오늘 들으실 전문용어가 다 들어있습니다."** — 명제 문장을 다시 읽으며 6개 용어가 각각 어느 구절인지 손가락으로 짚는 동작.
  4. **비유 가족(통일)**: "회계사무소 보조 인력" — LLM=말 잘하는 수석 비서 / 환각=기억 나는 대로 말해 버리는 실수(실존 조문인데 감면율·적용시기 살짝 틀림) / RAG=조문·예규·판례 더미에서 필요 부분을 찾아 비서 앞에 펼쳐 주는 손 / 코드=구석의 결정적 계산기(늘 같은 답) / 하네스=사무실 근태·결재 시스템(실행 틀)+그 위의 업무지시서·절차 매뉴얼(고삐) / 에이전트=심부름을 끝까지 수행하는 손발. *(이 비유 가족을 클로징까지 단일화. 마법·로봇·자율주행 비유 금지.)*
- **전환:** "그럼 이 두뇌(LLM)가 왜 가끔 틀리는지(환각)부터 보겠습니다." → M1 진입.
- **정직성 메모:** 에이전트/AX는 "맡기는 단계"로만. "지금 다 됩니다" 과장 금지(원천세 자동화만 구축, 감사 본류는 미구축). 이 4분에 **트랜스포머·파라미터·파인튜닝은 언급 금지** — 큰 그림은 map.

## 블록 M1 — AI의 원리: "가치 먼저, 그리고 왜 틀리는지" *(8분 — 환각은 캡처)*

- **목표:** AI가 *잘 하는 일*을 먼저 보여 가치를 주고, **환각(캡처)**으로 한계를 짚고, 원리로 설명합니다. 프롬프트 4요소+3강화는 슬라이드로만(실연은 후반부 비트①). **정규화 한 줄**로 뒤 1시간 가정을 충족합니다.
- **전할 한마디:** "AI는 언어를 다루는 데는 탁월합니다 — 긴 자료 요약, 흐름 정리, 초안 작성. 다만 *말로만* 숫자·사실을 꺼내면 틀립니다. 그래서 숫자는 계산기(코드)에, 사실은 검색에 맡겨야 정확해집니다."
- **보여줄 것 (순서대로):**
  1. **가치 먼저(1.5분)**: AI가 잘 하는 일 — 분개장 요약, 긴 이메일/회의록 3줄 압축, 조서 초안 골격. *"이런 건 정말 빠르고 쓸모 있습니다."*
  2. **환각(3분, 캡처본 — 라이브 아님)**: 
     - *A(빌드업)*: **실존하는 조문인데 감면율 수치·조문번호를 살짝 틀려** 인용(조세특례제한법 제7조 중소기업 특별세액감면) → '조용한' 환각. *"얼핏 보면 맞아 보이죠? 그래서 더 위험합니다."*
     - *B(결정타)*: **"조세특례제한법 제7조 감면은 2025.12.31.로 종료·폐지되었다"고 왜곡**(실제는 2028.12.31. 연장 현행) → *"이걸 납세자 감면 안내에 실었다면 받을 혜택을 놓치죠?"* 등이 서늘해지는 장면.
     - **대본(캡처 안내)**: "이건 제가 미리 캡처해 둔 겁니다 — 환각은 재현이 안 돼서요. **이걸 뒤 30분에 Claude Code 안에서 라이브로 다시 보여드리고, 어떻게 고치는지도 보여드리겠습니다.**" → 후반부 비트①의 복선.
  3. **원리(2분)**:
     - **뼈대**: "AI의 뼈대는 '말을 이어 가는 기계'에서 시작됐습니다 — 문맥을 읽고 가장 자연스러운 다음 말을 찾죠. 요즘은 그 위에 *생각 단계·계산기·검색 직접 쓰기·한 단계씩 계획 세우기*가 더해졌습니다. 다만 **뼈대 자체는 여전히 '말'이라**, 말이 아닌 것(정확한 숫자·검증된 사실)은 도구에 맡겨야 합니다."
     - **역설(명제와 연결)**: "반대로 AI가 *계산기·검색을 직접 쓰면* 환각이 크게 줄어듭니다 — 다만 검색 결과가 틀리거나 AI가 결과를 잘못 읽을 수 있어 완전히 사라지는 건 아닙니다(근거 왜곡). 그래도 '말로만' 물어볼 때보다 훨씬 안전합니다."
     - **★ 정규화 한 줄(pilot 가정 충족)**: "그래서 **숫자·사실은 도구에 맡기는데 — 숫자를 로컬 코드에 맡기려면, 데이터를 표준 틀로 한 번 맞춰(정규화) 그 위에서 결정적 분석을 돌릴 수 있습니다.** 그게 뒤 1시간의 뼈대입니다." (pilot "정규화가 토대" 가정을 이 한 줄로 충족.)
     - **계산기(결정적) ↔ AI('가장 자연스러운' 답 → 정답과 다를 때가 환각)**. *"확률적으로 찍는다"가 아니라 "가장 자연스러운 답을 고르다 보니 틀릴 수 있다."*
  4. **프롬프트 4요소 + 3강화(1.5분, 슬라이드만)**: 역할·맥락·지시·형식 위에 — ⑤ 예시(few-shot) ⑥ "단계별로 생각해" ⑦ "모르면 모른다고 해/근거 없으면 추측하지 마". *"이 일곱 가지, 특히 예시 하나만 붙여도 결과가 다릅니다 — 이건 뒤 30분에 직접 보여드립니다."*
- **정직한 단서(Q&A용):** *최신 모델은 코드·검색을 쓰면 환각이 대폭 줄지만 "완전히 사라진다"는 아닙니다(검색 결과 오염·근거 왜곡 가능). 가장 위험한 건 "말로만" 답을 꺼낼 때입니다. 그래서 "AI가 한 말"이 아니라 "AI가 띄운 후보"로 다루는 습관이 핵심입니다.*

## 블록 M3 — AX(개념): "챗봇을 넘어, 에이전트 시대" *(7분, 개념 — 라이브는 후반부로 이월)*

- **목표:** AX(Agentic Experience) 개념을 "UX → AX"로 설명하고, 하네스(고삐 2개)를 **정의만** 하고(실제 만들기는 후반부 비트②③), 회계 분업선을 그립니다.
- **전할 한마디:** "방금 '매번 설명하기 귀찮다'는 갈증을 짚었죠. 그 답답함을 풀어주는 방향으로, 요즘 소프트웨어가 한 단계 넘어갑니다 — 그걸 **UX에서 AX로** 넘어간다고 부릅니다. UX는 제가 버튼을 누르면 반응하는 것, AX는 **'이 일 끝까지 가져와 줘' 하면 AI가 알아서 찾고·모으고·정리해 초안까지 가져오는 것**입니다. 챗봇이 아니라, 일을 끝까지 가져와 주는 보조 인력을 두는 거죠."
- **보여줄 것(개념 슬라이드 중심):**
  1. **UX vs AX 비유 한 장**: 챗봇 = 내가 매 단계 누르는 계산기 / 에이전트 = 한 번 지시하면 자료 취합→정리→초안까지 가져오는 보조 인력.
  2. **하네스 — 실행 틀 + 그 위의 고삐 2개(정의만, 실제 만들기는 후반부)**: "하네스는 에이전트가 돌아가는 실행 틀입니다(도구·가드레일 포함). 그 틀 위에서 회계사님이 직접 만지실 고삐는 딱 두 가지 — **① 업무지시서(CLAUDE.md), ② 절차 매뉴얼(Skill)**. **이 두 고삐를 뒤 30분에 제가 직접 만들어 보여드립니다.** *별개로* 역할 분담(서브에이전트)·더존·홈택스 연결(MCP)도 있는데 '이런 것도 있다' 정도로만."
  3. **회계 분업선 — 한 장**: AX가 가져갈 자리 = **수임처별 대사(tie-out), 월결산 마감 진척 수집, 조회서 PDF 처리·집계, 조서 템플릿 채움**(규칙은 있되 매번 손으로 엮어야 하는 일) → 회계사가 가져갈 자리 = **판단·의심·결론·서명**.
  4. **결과물 1장**: "이 고삐를 세팅하면 *이런 결과물*이" — 회계 도메인 결과물 정지화면 1장(예: 세법 질의 Skill로 찰랑한 목록). *"직접 만드는 건 지금부터(후반부) 보여드립니다."*
- **전환:** "그럼 이 고삐 두 개를 직접 만들어 보겠습니다 — 제 컴퓨터에서 Claude Code라는 도구를 켭니다." → 후반부 진입.

## 블록 M4 — 신뢰와 책임: "한계를 아는 도구를 쓴다" *(7분, 압축)*

- **목표:** 환각 방지·보안·책임·한계 인정을 솔직하게 짚어, "이 사람은 과장하지 않는다"는 신뢰를 확보합니다. 회계사는 **한계를 아는 사람**에게 신뢰를 줍니다.
- **전할 한마디:** "AI가 다 해주진 않습니다. 하지만 **쓰는 동료와 안 쓰는 동료의 격차가 1~2년 안에 벌어질 것**입니다 — 업계 설문 기준 절반 가까이가 이미 매일 쓰고 있으니까요. 중요한 건, AI가 띄운 건 '의심'이지 '결론'이 아니라는 것, 그리고 최종 결론·서명은 언제나 회계사라는 것입니다."
- **보여줄 것:**
  1. **환각 방지(원리 재강조·1분)**: *말로만* 숫자·사실을 꺼내면 틀리니까 → 숫자는 코드, 사실은 검색에. (뒤 1시간에서 이 원칙이 어떻게 지켜지는지 예고.)
  2. **보안 — "접어두고"가 아니라 "푼 방식"(3분)**: "비밀유지의무 때문에 막막하시죠. 그런데 풀 수 있습니다 — **한 줄로: 숫자·계산은 전부 로컬 코드라 밖으로 나가지 않고, AI에 보내는 건 비식별 최소 정보, 더 민감하면 사내 로컬 모델 + 무보존(무학습·무저장) 계약.** 이게 뼈대입니다." + **"다만 이건 피해갈 수 없다" 3종(상세는 별도 검토)**: 재식별 위험(사업자번호+금액+귀속월이면 식별) / 미공시정보(상장·코스닥 수감회사 결산 자료는 로컬 강제) / 감리·증적 보존(AI 산출물도 감사증적으로 보존). *"이 3종은 별도 정보보안·컴플라이언스 검토가 필요합니다 — 오늘은 '알아둘 위험'으로만, 상세는 유료에서."* **(이 보안 뼈대는 후반부 Claude Code 데모의 전제 — 비트②③은 비식별 더미 데이터 사용.)**
  3. **"기계가 하는 일 vs 사람이 하는 일" 한 장(1.5분)**: 줄 단위 명시(취합·정리·초안 = 기계 / 판단·의심·결론·서명 = 사람).
  4. **책임소재 한 줄(1.5분)**: "AI가 만든 산출물의 최종 책임은 서명한 회계사에게 있습니다 — 그래서 회계사의 자리가 사라지지 않습니다."
- **전환:** "개념은 여기까지. 이제 **제가 직접 도구를 켜고 보여드리겠습니다**." → 후반부.

  - **감사 본류(위험평가·조서 자동작성)·원천세 워크플로우는 후반부 라이브에서 시연 금지** — 미구축/유료 영역.
  - 터미널·코드 노출 최소, "코드가 아닌 한국어 업무지시서" 강조.

# 예상 질문 대응

| 나올 질문 | 답변 요지 |
|---|---|
| **"산수·사실은 못한다고 했는데, 뒤에선 AI가 공시 검색·분석한다며? 모순 아닌가?"** | 정확히 짚으셨습니다. *말로만* 꺼내면 틀리지만, 코드로 계산하고 검색으로 사실을 가져올 때는 훨씬 정확해집니다(단, 완전 무결은 아닙니다 — 검색 결과 오염·근거 왜곡 가능 — 그래서 회계사의 최종 검산이 남습니다). 뒤 시연은 숫자는 코드에, 사실은 검색에 맡긴 형태로 설계돼 있습니다. |
| **"AI가 숫자를 확률적으로 찍는다고요? 감사 숫자도 찍나?"** | '확률적'은 '대충 찍는다'가 아니라 '가장 자연스러운 다음 말을 고른다'는 뜻입니다. 숫자 부분은 코드로 돌려 AI가 손대지 못하게 해, 찍는 일이 아예 안 일어나게 설계합니다. |
| **"46%라는 숫자, 출처가 어디? 공식 통계인가?"** | 업계 설문(FindSkill.ai, 2026) 기준이고 공식 통계는 아닙니다. 정확한 수치보다 '이미 대세'라는 점이 중요해서 인용했습니다. |
| **AI가 숫자를 지어내지 않나? (환각)** | 맞다. 그래서 *말로만* 꺼낸 숫자·사실은 안 쓰고, 코드·검색에 맡긴다. AI가 띄운 건 '의심'이지 '결론'이 아니다. |
| **매번 결과가 다른데 믿고 쓰나?** | 언어 부분은 그래서 들쭉날쭉. 그래서 하네스의 고삐(CLAUDE.md·Skill 두 개)로 내 기준을 고정합니다 — 방금 보여드린 것처럼요. 전체 세팅은 다음 단계(유료)에서. |
| **비밀유지의무는? (공격적)** | 회피 안 한다. 숫자·계산은 로컬(미전송), AI엔 비식별 최소, 로컬 모델 옵션, 무보존 계약으로 양립. 단, 재식별·미공시정보·감리 대응은 별도 검토 필요 — 상세는 유료에서. |
| **책임소재는?** | AI 산출물의 최종 책임은 서명한 회계사. 그래서 판단·서명 자리가 사라지지 않는다. |
| **클로드 코드, 설치 어렵지 않나? (비개발자)** | 오늘은 강사가 직접 보여드립니다. CLAUDE.md·Skill은 한국어로 쓰는 거라 코딩이 아닙니다. '내 회계실에 세팅'은 유료에서 비개발자도 따라 하도록 단계화합니다. |
| **이걸로 감사 시스템 다 되나? (과장 의심)** | 아닙니다. 오늘은 도구(Claude Code) 쓰는 법을 보여드린 겁니다. 완성된 감사 시스템(위험평가·조서·정산표)은 지금부터 뒤 1시간에, 그리고 전체 세팅은 유료에서 다룹니다. 세팅 자체는 학습이 필요합니다. |
| **유료 과정 기간·비용·결과물은?** | (안내 데스크·슬라이드) 회계 특화 결과물 3종 + 기간(예: 5주 평일 저녁) + 평생소장. 가격은 안내 자료 참고. |
| **빅4 상용 감사소프트웨어와 다른 점?** | 오늘은 '원리·실사용'에 집중. 도구 비교는 별도 자료로 안내. |
| **감사품질관리·증적·표본추출은?** | 오늘(원론) 범위 밖. 뒤 1시간(실연) 기획에서 별도로 다룬다. |

# 업셀링 설계 (무료 → 유료)

- **벤치마크**: 스파르타 클로드코드 과정(비개발자 5주, *"AI 팀을 굴리세요"*, CLAUDE.md·Skill·서브에이전트·MCP·하네스, **무료 사례집으로 결제 유도**).
- **전략(v3.0 전환)**: "AX 실연을 미뤄 기대감 축적" → **"도구 입문은 직접 보여주되, 전체 시스템 세팅은 유료"**. 후반부가 "직접 보니 더 갖고 싶다"로 작동하되, 갈증이 완전 상쇄되지 않게 **세팅의 마지막 단계(수십 개 엮기·실데이터·보안)**를 남김.
- **무료 후반부가 보여주는 것(갈증 해소)**: CLAUDE.md 한 장 직접 만들기 / Skill 1개(`/세법질의`) 실행 / 환각→정정 단일 사이클 / 더미 데이터.
- **유료가 챙기는 것(남는 갈증)**: 수감회사별·업무별 CLAUDE.md 세트 / 회계 특화 Skill 팩(세법 질의·분개장 탐지·조회서 집계·조서 템플릿) / 환각 억제+검증 파이프라인(RAG 코퍼스 세팅) / 도구들을 엮은 워크플로우(원천세 end-to-end) / 본인 실데이터+보안 컴플라이언스.
- **핵심 한 줄**: *"한 번 해보는 것"은 무료, "내 회계실에 영구 세팅하는 것"은 유료.*
- **CTA 3회 미세 배치(v3.0)**: ① 비트① 끝(약·갈증 씨앗 "매번 다시 설명하시죠?") → ② 비트③ 끝(중·결과물 정지화면 "수십 개 세팅하면?") → ③ 클로징(강·오퍼 "내 회계실 AI 팀 세팅"). 강도 점진 상승.
- **갈증→구매 의사 전환**: v2.2.1은 M3 정지화면으로 전환했는데, **v3.0은 비트② CLAUDE.md에서 이미 "이거 나도 만들어 보고 싶다"는 전환이 일어남**(더 강력). 비트③ 정지화면은 그걸 확장.
- **유료 오퍼(도메인 특화·구체)**: ① 세법 질의답변 Skill 팩 ② 분개장 이상징후 탐지 에이전트 ③ 원천세 신고 워크플로우 세팅 문서. 기간(예: 5주 평일 저녁)·결과물("수료 시 내 회계실 AI 팀 1식 + 세법 질의 Skill 팩")·가격대는 안내 자료로.
- **pilot과의 퍼널 분업**: 원론 후반부="도구 입문"(개별 도구) / pilot="완성 시스템 실연"(감사 본류 결과물) / 유료="직접 내 회계실에 건설". 세 단계 분업.
- **전환 자산(강의장 밖 루프)**:
  1. **회계 도메인 사례집 PDF(리드 마그넷)**: "회계사 5인의 '내 AI 팀' 사례 5선" — 이메일/번호 남기면 전송(리드 확보).
  2. **'내 회계실 AI 준비도 체크리스트'**: 10문항 셀프 진단 → "AX 도입 시 O시간/월 절감 가능" 맞춤 메시지(갈증을 숫자로).
  3. **5일 메일/카톡 시리즈**: D+1 환각·CLAUDE.md 복습 → D+2 프롬프트·Skill 템플릿 → D+3 사례집 1편 → D+5 유료 마감 + 회계사 인터뷰. 강의장에서 결제 못 한 90%를 5일 안에 재터치.

# 미리 정해 두면 좋은 것 (가정 표시)

1. **청중**: 웹 ChatGPT·Claude는 써봤으나 코딩 경험 없는 현직 회계사. (세무사 동반 시 예시 비중 조정.)
2. **형식(v3.1)**: **전반부 30분 PPT 개념 + 후반부 30분 Claude Code 라이브(강사 시연)**. 핸즈온은 기본 없되, **비트② 끝 폰 따라하기(CLAUDE.md "한 줄 라벨" 복붙)는 기본 진행**(v3.1 — 능력 체감, 폰 데이터로 가능).
3. **AX 범위(v3.1)**: 원론 후반부에서 **CLAUDE.md 만들기(체험)**·**Skill 실행만**을 직접 실연. **Skill 만드는 법·서브에이전트·MCP는 시연하지 않음**(클로징 멘트 1줄로만) — 유료 영역. **완성된 감사 시스템(위험평가·조서·정산표)·원천세 워크플로우도 시연 안 함** — 뒤 1시간(pilot)과 유료(전체 세팅)로.
4. **보안**: "접어두고"가 아니라 "푼 방식"으로 — 숫자=로컬, AI엔 비식별 최소 + 무보존. 재식별·미공시·감리증적은 한 줄씩만, 상세는 유료에서. **후반부 Claude Code 데모는 비식별 더미 데이터 사용.**
5. **기술 정확성**: "산수·사실은 못한다"가 아니라 "*말로만* 꺼내면 틀리니까 도구에 맡긴다"로 — 뒤 1시간(정규화·코드 계산·검색)과 모순 없게.
6. **도구 입문은 보여주되 세팅은 유료**(v3.0)로 업셀링 동력 설계 — "한 번 해보는 것"은 무료, "영구 세팅"은 유료.
7. **뒤 1시간**(`pilot_lecture_plan.md`)과 톤·템플릿·신뢰 프레임을 일치시켜 한 시리즈로 보이게 함. pilot은 "정규화가 토대"를 가정 → 원론 M1 정규화 한 줄로 충족.
