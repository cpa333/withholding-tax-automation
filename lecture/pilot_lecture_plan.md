# 실무강의 — 커리큘럼과 진행 대본 (초안 v8)

> 회계감사 AX 실무강의(60분)를 위한 기획서입니다. 원론강의 1시간이 먼저 진행된 뒤,
> 이 60분에서는 **완성된 시스템을 실제로 돌려 보여주는 방식**으로 시연합니다.
> 데이터는 전부 **가상의 회사(ABC상사)**로 만들어, 보안 부담 없이 통제된 형태로 시연합니다.
>
> **이 강의가 노리는 것:** "이게 실제로 된다"는 체감과 도입 동기. 현직 회계사 대상이므로
> *감사 절차 자체를 설명하지 않고*, 그 절차가 **어떻게 자동화·증강(AX)되는지**(지금 손으로 하던 일 → 자동)에만 집중합니다.
> 청중은 ChatGPT·Claude는 써봤지만 코딩은 안 해본 회계사입니다. 화면도 문서도 코드보다 결과 위주로 갑니다. (작성 기준일: 2026-06-16)

---

## 오늘의 핵심 명제

> **감사 자동화의 핵심은 화려한 AI가 아니라, 고객의 지저분한 엑셀을 한 번 제대로 *정규화*해서
> 그 위에서 분석을 *결정적으로* 돌릴 수 있게 만드는 것입니다.**

뒤에 나오는 장면들 — 위험을 자동으로 짚는 분석, 조서가 채워지는 것, 정산표가 만들어지는 것 — 은 전부
*맨 처음 한 번의 정규화* 위에서 가능해집니다. 데이터를 표준 틀로 한 번 맞춰 두면, 같은 데이터가
위험평가·조서·정산표에서 *그대로* 쓰입니다. 강의는 이 점을 중심축으로 둡니다.

---

## 강의의 큰 흐름 — 두 묶음

- **1막. 감사 본류** — 고객 엑셀을 한 번 정규화하면, 그 데이터 위에서 위험을 평가하고 조서(리드시트)로 정리하기까지의 흐름.
- **2막. 실무 도구들** — 그 정규화한 데이터가 이어지는 도구(정산표·DSD)와, 감사 곳곳에 흩어진 독립적 실무 도구들(PDF·제안서·복합상품·원천세).

1막과 정산표·DSD는 **같은 정규화 데이터 한 벌을 공유**합니다 — 한 번 정규화하면 그 데이터가 위험평가·조서·정산표·DSD로 그대로 흐릅니다.
이 연결을 강의에서 화면으로 보여줍니다. 반면 그 뒤 도구들(PDF·제안서·복합상품·원천세)은 각각 독립적인 실무 작업입니다.

---

## 한눈에 — 60분 배분

| 막 | 순서 | 시간 | 비중 |
|---|---|---|---|
| **1막. 감사 본류** (28분) | ① 자료 정규화 — *모든 것의 토대* | 03:00–07:00 | 4분 |
| | ② 위험평가 — *결정적 분석을 실제로 실행* | 07:00–19:00 | **12분 (중심)** |
| | ③ 조서로 정리 — 리드시트 + 위험결과 + 절차 소개 | 19:00–31:00 | **12분 (중심)** |
| **2막. 실무 도구들** (26분) | ④ 정산표 만들기 *(데이터가 이어짐)* | 31:00–36:00 | 5분 |
| | ⑤ DSD 파싱·편집 *(정산표에서 이어짐)* | 36:00–41:00 | 5분 |
| | ⑥ PDF 파싱 (금융기관 조회서) | 41:00–45:00 | 4분 |
| | ⑦ 제안서 작성 (AI) | 45:00–49:00 | 4분 |
| | ⑧ 복합상품 평가모델 — 소개만 | 49:00–53:00 | 4분 |
| | ⑨ 원천세 자료 자동 출력 *(녹화)* | 53:00–57:00 | 4분 |
| — | 오프닝 03:00 · 클로징 03:00 | 00:00 / 57:00 | — |

- 1막(본류) 28분과 2막(도구) 26분이 거의 반반입니다.
- **결정적 분석을 *실제로 돌려* 보여주는 장면은 위험평가(②) 하나**입니다 — 핵심 명제를 여기서 증명합니다.
  조서(③)의 실증절차는 *실제 수행까지 가지 않고* "리드시트 + 위험결과 + 절차 소개"까지, 다만 *이 절차들도 자동화할 수 있음*을 함께 언급합니다.
- **시간이 모자라면 덜어내는 순서:** 복합상품(⑧) → 제안서(⑦). 원천세(⑨)는 실무 가치가 커서 가급적 지키고, 본류의 두 중심(②·③)과 정산표(④)는 끝까지 지킵니다.

# 1부. 커리큘럼 — 꼭지별 설계

> 각 꼭지를 *목표 / 청중에게 전할 한마디 / 무엇을 보여줄지* 순으로 정리했습니다.
> 전할 한마디는 모두 **"지금 손으로 하던 일 → 자동"**의 차이에 초점을 둡니다(감사 절차 자체를 설명하지 않음).

## 1막. 감사 본류

### 오프닝 — "엑셀 한 번 정규화하면, 그 데이터가 하루 전체를 굴린다"
- **목표:** 핵심 명제(정규화가 결정적 분석의 토대)와 두 묶음 구성을 알립니다.
- **전할 한마디:** "오늘 하나만 남기신다면 — 자동화의 핵심은 AI가 아니라 데이터 정규화입니다. 그 위에서 위험평가도, 조서도, 정산표도 결정적으로 돌아갑니다. 지금부터 그걸 실제 시스템으로 보여드리겠습니다."
- **전제 한마디(보안, 30초):** "한 가지만 미리 — 오늘은 *보안은 접어두고* LLM을 자유롭게 쓴다고 가정합니다. 다만 비밀유지의무 때문에 막막하실 텐데, 실무에선 풀 수 있는 문제예요. *숫자·계산은 어차피 전부 로컬 코드라 밖으로 나가지 않고*, AI에 보내는 건 4개 부분(계정 매핑·예외 설명 등)의 *비식별 최소 정보*뿐입니다. 더 민감하면 사내 로컬 모델을 쓰고, 거기에 고객 동의·무보존(무학습·무저장) 계약을 더하면 비밀유지의무와 양립합니다. 오늘은 여기 머물지 않고 'AX가 무엇을 할 수 있나'에 집중하겠습니다."
- **보여줄 것:** 시스템 첫 화면, 가상 고객 ABC상사 소개 한 장, 오늘의 지도 한 장.

### 꼭지 ① — 자료 정규화: "여기가 모든 것의 토대다" *(4분, 빠르게)*
- **목표:** 정규화의 값어치가 *합계 맞추기*가 아니라, **제각각인 엑셀을 표준 틀로 맞춰 이후 분석을 결정적으로 가능케 하는 것**임을 분명히 합니다.
- **전할 한마디:** "차대 합계 맞추기야 일도 아니죠. 정규화의 값어치는 거기가 아닙니다. 고객마다 제각각인 엑셀을 *하나의 표준 틀*로 맞춰, 그 데이터가 위험평가·조서·정산표에서 그대로 쓰이게 하는 겁니다. (받은 자료가 모집단을 빠짐없이 담았는지 완전성은 짚고 갑니다 — 그게 어긋나면 위의 분석이 전부 헛것이 되니까요.)"
- **보여줄 것:** 제목·병합·소계로 어수선하던 엑셀 4종(재무상태표·손익계산서·분개장·원장)이 표준 표로 정렬되고, 계정과목에 표준 이름표가 붙는 모습. 완전성은 "1만 2천 건 빠짐없이 연결" 한 줄로.
- **정직한 단서(언급/Q&A용):** *형식이 고객마다 달라도 처리하지만, 새 형식은 한 번 "이 칸이 차변·날짜" 매핑을 잡아 주면 이후 자동 재사용입니다. 이 정규화 본체는 LLM 없이 결정적 코드로 구현됩니다(LLM은 계정 매핑을 제안해 거들 뿐, 필수 아님).*

### 꼭지 ② — 위험평가: "어디가 수상한지 추린다" *(12분, 중심·결정적 분석 실연)*
- **목표:** 정규화된 데이터 위에서 *정량 분석*(전당기·비율·추세·관계)이 실제로 돌고, *AI 산업분석*을 더해 **최종 위험평가**로 묶이는 걸 보여줍니다.
- **전할 한마디:** "정규화해 둔 덕에, 손으로 깎던 작년 대비·비율·월별 흐름·예측관계 분석을 1만 2천 건 위에서 한 번에 계산합니다. 그러면 '12월 매출 38% 급증', '매출은 정체인데 매출채권만 급증' 같은 후보가 추려져요. 여기에 **AI 산업분석을 붙입니다 — 회사·산업만 지정하면 공시자료 등은 AI가 알아서 검색해 반영**하고요. 정량 분석과 산업분석을 합쳐 *최종 위험평가*가 나옵니다."
- **보여줄 것:**
  1. 정량 분석 실행 → 위험 후보가 목록에 쌓이고 심각도(상·중·하) 자동. *(여기가 "결정적 분석을 실제로 돌리는" 핵심 장면.)*
  2. **AI 산업분석**: 회사·산업 지정 → AI가 공시 등을 검색·반영해 산업 위험 초안 생성.
  3. 정량 + 산업분석이 **하나의 최종 위험평가**로 묶이고, 회계사가 후보를 채택/기각(기각은 사유)하는 모습. "AI가 띄운 건 의심이지 위험이 아닙니다 — 채택은 회계사."
- **전환:** "이 결과를 조서로 정리해 봅니다."

### 꼭지 ③ — 조서로 정리하기: "리드시트 + 위험결과 + 실증절차" *(12분, 중심)*
- **목표:** 정규화된 데이터로 *리드시트가 모든 계정과목에 자동으로 작성되고*, 위험평가 결과가 얹히며, 계정별 실증절차가 정리된 **엑셀 조서철**이 만들어지는 걸 보여줍니다. 감사 절차를 설명하는 게 아니라, 그 산출을 자동화하는 모습입니다.
- **전할 한마디:** "리드시트는 모든 계정과목에 자동으로 그려집니다. 거기에 위험평가 결과가 관련 계정에 메모로 얹히고, 계정별 실증절차가 정리됩니다. 산출물은 엑셀(필요하면 워드)이고요. 오늘은 시간상 절차를 실제로 *수행*하는 데까진 안 가고 정리·소개까지만 보여드리는데 — 짚어 둘 건, **이 계정별 실증절차의 수행까지도 파이프라인에 넣을 수 있다**는 점입니다. 회계사가 쥐는 건 결론·판단·서명이고요."
- **보여줄 것:**
  1. 정규화된 잔액으로 **모든 계정과목 리드시트 자동 작성**(당기·전기·증감), 엑셀 형태.
  2. 관련 계정에 **위험평가 결과**가 메모로(예: 매출채권 리드시트에 "매출 정체 대비 채권 급증").
  3. 그 아래 **계정별 실증절차 정리** + "이 절차들의 수행도 자동화로 포함 가능"을 한 줄.
  4. 결론 문장은 *AI 초안*까지 — 확정·서명은 회계사.
- **포인트(만드는 방식):** 숫자·표·리드시트는 **코드가** 채웁니다(AI 접근 불가 → 환각 차단). 전체 조립과 서술 초안은 **AI가 지휘**합니다. 그래서 숫자는 지어낼 수 없고, 같은 입력이면 같은 조서가 나옵니다. (인터뷰의 '퀄리티 보장' 우려 대응.)

## 2막. 실무 도구들

> **넘어가는 한마디:** "여기까지가 감사 본류였습니다. 이제 그 주변 실무 도구들인데, 두 종류예요 — ④정산표·⑤DSD는 *방금 정규화한 데이터가 그대로 이어지는* 도구이고, 그 뒤(⑥~⑨)는 각각 독립적인 실무 도구입니다. 감사 곳곳에 흩어진, 그렇지만 자주 손이 가는 일들이죠."

### 꼭지 ④ — 정산표 만들기 *(정규화 데이터가 이어짐, 5분)*
- **목표:** 방금 정규화한 데이터가 감사 본류 말고 실무 산출물로도 곧장 쓰임을 보여 줍니다.
- **전할 한마디:** "같은 데이터로 정산표를 자동 세팅합니다 — 계정별 잔액, 수정분개 칸, 재무제표 연결까지. 그리고 이 정산표 숫자가 바로 다음 DSD로 들어갑니다."
- **보여줄 것:** 정규화된 잔액 → 정산표 양식 자동 채움.
- **메모:** 다음 DSD로 이어지는 연결고리.

### 꼭지 ⑤ — DSD 파싱·편집 *(정산표에서 이어짐, 5분)*
- **목표:** 공식 감사절차는 아니지만 실무에서 매년 다루는 DSD 작업을, 앞 정산표와 이어 보여줍니다.
- **전할 한마디:** "DSD에서 세 가지를 보여드립니다 — ① 당기 숫자를 전기로 이월, ② 방금 만든 정산표 숫자를 DSD 재무제표 칸에 입력, ③ 별개로, *같은 문서의 구버전과 신버전을 비교*해 바뀐 곳을 짚어 줍니다."
- **보여줄 것:** ① 당기→전기 이월 → ② 정산표(④) 숫자가 DSD 재무제표로 자동 입력 → ③ (별개 기능) 구버전 vs 신버전 변경 비교.

### 꼭지 ⑥ — PDF 파싱 (금융기관 조회서) *(4분)*
- **목표:** 종이·스캔 PDF를 엑셀로, 실무 편의 기능까지.
- **전할 한마디:** "조회서 PDF를 엑셀로(숫자에 쉼표까지), 도장 일괄 날인, 금액 옆 체크 표시, 회사별 폴더 자동 분류."
- **보여줄 것:** PDF 투입 → 엑셀 추출 + 도장·체크 + 폴더 분류(라이브 또는 캡처).

### 꼭지 ⑦ — 제안서 작성 (AI) *(4분)*
- **목표:** 기획은 글로, 디자인은 슬라이드로 — 제안서를 빠르게.
- **전할 한마디:** "감사·비감사 제안서를, 먼저 글로 기획을 잡고 슬라이드로 바꿉니다. 웹 AI로 하시던 걸 틀로 체계화한 거예요."
- **보여줄 것:** 요청 → 기획 글 초안 → 슬라이드 변환(짧게).

### 꼭지 ⑧ — 복합상품 평가모델 *(소개만, 4분 / 가장 먼저 덜어낼 후보)*
- **목표:** 전문가용 영역도 직접 만들 수 있음을 맛보기로.

# 3부. 강의 전에 준비할 것

## 준비물 1 — 시연용 시스템 (지금은 설계만, 강의 전 제작 필요)

지금 이 시스템은 **설계 문서만 있고 아직 만들지 않은 상태**입니다. 강의 전에 *시연에 필요한 최소한*만 만들면 됩니다.

| 화면 | 무엇을 받아 | 무엇을 내놓나 | 만들기 |
|---|---|---|---|
| 정규화 | 원본 엑셀 4종 | **표준 데이터 한 벌**(이후 전부의 토대) + 완전성 한 줄 | 꼭 필요(토대) |
| 수상한 후보 목록(정량) | 표준 데이터 | 위험 후보·심각도·근거 | 꼭 필요(중심) |
| AI 산업분석 | 회사·산업 지정 (공시 등은 AI가 검색) | 산업 위험 초안 → 최종 위험평가로 통합 | 결과 미리 저장 권장 |
| 조서: 리드시트+위험+절차 | 표준 데이터 + 위험 후보 | 전 계정 리드시트·위험 메모·실증절차가 담긴 **엑셀 조서철** | 꼭 필요(중심) |
| 정산표 | 표준 데이터(잔액) | 정산표 파일 *(→ DSD로 연결)* | 꼭 필요(연결 증명) |
| DSD | 정산표 산출 + DSD 문서(구·신버전) | 당기→전기 이월·정산표 입력·버전 비교 결과 | 라이브 |
| 그 밖 도구(PDF·제안서·복합상품·원천세) | 각자 입력 | 각자 결과 | 일부 라이브·일부 녹화 |

**연결성·구현 가능성.**
- **한 줄기로 이어짐:** '정규화'가 만든 *표준 데이터 한 벌*을 위험평가·조서·정산표·DSD가 그대로 읽습니다.
- **재현성:** 숫자 부분은 전부 결정적 코드라, 같은 입력이면 늘 같은 결과가 나옵니다(리허설=본방).
- **조서 생성(혼합):** 숫자·표·리드시트는 코드가 채우고(AI 접근 불가), 조립과 서술 초안만 AI가 지휘합니다.
- **만드는 순서:** 정규화 → 후보 목록 → 조서 → 정산표 → AI 산업분석(저장본) → 나머지 도구. 화면은 코드보다 결과 위주(청중 코딩 미경험).

## 준비물 2 — 가상 회사 데이터 (ABC상사)

위험평가에서 *후보가 떠오르는 장면*이 나오도록, 통제된 이상 항목을 심어 둡니다.

| 심어 둘 이상 항목 | 어디서 떠오르나 |
|---|---|
| 매출은 정체인데 매출채권만 급증 | 위험평가(관계분석) → 조서(매출채권 리드시트 위험 메모) |
| 12월 매출이 작년보다 38% 급증 | 위험평가(월별 흐름) |
| 결산일에 분개가 몰림 | 위험평가(분개 수준 요약 신호) |
| 신규·일회성 거래처 | 위험평가(분개 수준 요약 신호) |
| 같은 금액 중복 지급 의심 2건 | 위험평가(분개 수준 요약 신호) |

- 분개장 1만 2천 건 규모, 작년(전기) 숫자·적요·거래처 보유. **완전성은 '통과'로 설계.**
- **데이터는 의도적으로 통제합니다**(아래 '예상 질문' 참조): 실데이터처럼 엉망이면 예외가 쏟아져 시연이 성립하지 않습니다.
- DSD·조회서 PDF·정산표용 부속 파일도 전부 ABC상사 이름으로.

## 준비물 3 — 예상 질문 대응

| 나올 질문 | 답변 요지 |
|---|---|
| **중소기업은 회계처리가 엉망이라 예외가 쏟아지지 않나?** | 맞다, 실제 위험이다. 그래서 ① 중요성 미달 잡음은 거르고 ② 단일 룰이 아니라 다중 기준 결합으로만 후보가 되며 ③ 예외는 '결론'이 아니라 *우선순위 신호*라 심각도순으로 추린다. 시연을 통제된 가상데이터로 하는 이유이기도 하다. |
| 아무 엑셀이나 다 되나? | zero-config는 아니다. 새 형식은 1회 매핑 후 자동 재사용. |
| AI가 숫자를 지어내지 않나? | 숫자 칸은 코드 전용, LLM 접근 불가. AI는 조립·서술만. |
| 정규화도 결국 AI 의존 아닌가? | 본체는 결정적 코드. LLM 없이 작동(매핑 제안만 보조). |

## 준비물 4 — 시연 사고 대비

| 위험 | 미리 | 그 자리에서 |
|---|---|---|
| AI·웹 검색 지연·실패 (가장 큼) | 산업분석·결론 초안 결과를 **미리 저장** | 저장본 띄우고 "실제론 30초~1분" |
| 엑셀 로딩·실행 실패 | 단계별 **결과 파일** 보관 | 그 파일 열어 설명 |

| 시간 초과 | 덜어낼 순서(복합상품→제안서, 원천세는 지킴) | 해당 꼭지 생략하고 자료 공유 |

# 4부. 미리 정해 두면 좋은 것 (제 가정 표시)

1. **시간 비중:** 본류 28분 : 도구 26분(위험평가 12 + 조서 12). 더 줄이려면 조서를 10분으로 당길 수 있습니다.
2. **결정적 분석 실연은 위험평가 한 장면.** 실증절차는 실제 수행 없이 *소개*하되, "수행도 자동화 가능"을 함께 언급(감사 절차 설명이 아니라 자동화 관점 유지).
3. **AI 산업분석 입력 = 회사·산업 지정**(공시 등은 AI가 검색·반영).
4. **조서 산출물 = 엑셀(필요 시 워드).** 생성은 혼합(숫자=코드, 조립·지휘=AI). *출처 추적(수치 되짚기)은 1:1 대응이 불가능해 넣지 않음. 클릭형 뷰어 앱도 만들지 않음.*
5. **2막 구성:** ④정산표·⑤DSD는 정규화 데이터가 이어지는 도구, ⑥~⑨는 독립적 실무 도구. 덜어낼 순서는 복합상품→제안서(원천세는 지킴).
6. **원론강의에서 어디까지 다뤘는지** 알려 주시면 개념 재설명 분량을 조정합니다(지금은 '정규화가 토대' 개념을 이미 봤다고 가정).
7. **보안 전제:** 이 강의는 보안을 접어두고 LLM을 자유롭게 쓴다고 가정합니다. 해소 방안(결정적 엔진은 로컬·외부 미전송 / LLM엔 비식별 최소 정보 또는 사내 로컬 모델 / 고객 동의·무보존 계약)은 *서두에 30초로만* 짚고 상세는 다루지 않습니다. 근거: 보안정책 문서(`09`).
