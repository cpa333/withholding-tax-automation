## (A) 전역 디자인 토큰

> 모든 슬라이드·모든 템플릿에 공통. codex는 이 토큰을 상수로 두고, gpt-image-2 프롬프트와 python-pptx 양쪽에 동일 값을 주입한다.

### A-1. 컬러 팔레트 (오렌지 단색 — Nexus 충실 매핑)

| 토큰 | hex | 용도 |
|---|---|---|
| `BG` | `#FFFFFF` | 슬라이드 배경(순백) |
| `INK` | `#1A1A1A` | 제목·빅스테이트먼트(검정) |
| `BODY` | `#4A4A4A` | 본문 텍스트(회색) |
| `MUTE` | `#6B7280` | 보조·캡션·출처(연회색) |
| `LINE` | `#E5E5E5` | 구분선·표 테두리·카드 테두리(연회색) |
| **`ACCENT`** | **`#FF4D30`** | **단일 강조색 — 로고 점·블록 라벨·빅넘버·데이터카드 테두리·절차번호·강조 밑줄·현재 위치 표시** |
| `ACCENT_SOFT` | `#FFF1ED` | 오렌지 8% 채움(카드 배경·활성 행 하이라이트) |

- **규칙:** 한 슬라이드에 ACCENT는 **강조 자리에만**. 본문 도배 금지. 회계사 기획서의 앰버(환각경고)·세이지(사람)는 **전부 ACCENT 오렌지로 통일**(단색 정책).
- **금지:** 그라데이션·다크모드·드롭섀도우·사진·3D.

### A-2. 타이포그래피 (한글 폰트 중심)

> Nexus 원본은 Bebas Neue(표지)/Montserrat(라벨)/Lato(본문). 한글은 압축 display 대응이 없으므로 **Pretendard 계열**로 매핑. 영문 라벨·숫자는 Pretendard 또는 Inter로 동일 톤 유지.

| 토큰 | 폰트 | 용도 |
|---|---|---|
| `KR_BODY` | Pretendard Regular | 한글 본문 |
| `KR_BODY_MED` | Pretendard Medium | 본문 강조 |
| `KR_TITLE` | Pretendard Bold | 슬라이드 제목(H1) |
| `KR_DISPLAY` | Pretendard ExtraBold | 표지·명제 빅타이포 |
| `LABEL` | Pretendard Bold(또는 Inter Bold) | 영문 대문자 라벨, 자간 넓게(+2~3pt) |
| `NUMERIC` | Pretendard ExtraBold/Black | 빅넘버(숫자) |

### A-3. 캔버스 & 그리드

- **슬라이드:** 16:9 = **13.333in × 7.5in**(= 1920×1080px @144dpi 기준).
- **안전 여백:** 좌우 **6%**(= 0.80in / X=6~94%), 상 **7%**(Y=7~), 하 **7%**(Y~93). 이 안쪽이 본문 영역.
- **좌표 규약:** 본 문서의 모든 좌표는 **(X%, Y%) = 좌상단, W% × H% = 크기**(슬라이드 전체 100×100 기준).
- **python-pptx 환산:** `Inches(X/100 × 13.333)`, `Inches(Y/100 × 7.5)`.
  - 예: (10, 12, 60, 6) → `add_textbox(Inches(1.33), Inches(0.90), Inches(8.00), Inches(0.45))`.
- **정렬 기준:** **좌측 정렬 우선**(Nexus 시그니처). 중앙 정렬은 명제·표지·캡션 슬라이드만.
- **1 슬라이드 1 메시지.** 본문 텍스트 **5줄 이내**(캡처 슬라이드 제외).

- **에디토리얼·스위스 미니멀.** 넉넉한 여백, 텍스트 중심.

## (B) 레이아웃 템플릿 9종 (17장을 커버하는 재사용 단위)

> 각 템플릿 = (1) 영역 배치도 (2) 텍스트박스 표 (위치·크기·폰트·정렬·색·**한글 권장 자수**) (3) **gpt-image-2 담당**(도형 골격) (4) **python-pptx 담당**(한글 텍스트).
> **한글 권장 자수** = 그 박스 1줄의 안전 한글 자 수(공백 제외). 공식: `1줄 자수 ≈ 박스폭(inch) × 72 ÷ pt × 0.82`.

## (D) gpt-image-2 마스터 프롬프트 (모든 템플릿에 공통)

> codex가 각 템플릿(T1–T9) 이미지를 생성할 때 **반드시 아래 시스템 접두사를 앞에 붙이고**, 뒤에 템플릿별 도형 묘사(B절)를 이어 붙인다.

```
[SYSTEM — 모든 템플릿 공통]
A minimalist editorial slide-layout TEMPLATE (background skeleton only), 16:9 aspect ratio, 1920×1080.
Style: Swiss/editorial minimalism. Pure white background (#FFFFFF). Heavy whitespace.
Single accent color: orange #FF4D30 (use sparingly — only for: one logo dot, section-label underline, big-number slot frame, step numbers, active-node fill, 1px card borders where specified). Neutral grays: text slots #4A4A4A, hairlines #E5E5E5.
ABSOLUTELY NO: gradients, dark mode, drop shadows, 3D, photos, illustrations, emojis, or icons.
CRITICAL — TEXT POLICY: This is a LAYOUT SKELETON. Render NO readable text. Leave every text region as a clean empty slot (a subtle dashed/light rectangle or just blank space). Do NOT attempt to render Korean characters or any sentence text. Only empty labeled slots are allowed.

[TEMPLATE-SPECIFIC GEOMETRY — B절의 도형 묘사를 여기에]
예(T4 개념지도): "Central large circle (orange 1px border, no fill) at center; five smaller rectangles/circles arranged radially around it (top-left, bottom-left, top-right with orange border, bottom-center wide, bottom-right), connected by thin #E5E5E5 lines; a thin orange-tinted bar across the top. All nodes are empty slots."

Output: a clean, print-ready layout skeleton image, 16:9.
```

**codex 지시(gpt-image-2 호출 시):**
- 모델: `gpt-image-2`(또는 사용 가능한 최신 이미지 모델).
- 크기: `1536×1024`(가로) 또는 `1792×1024` — 16:9에 가장 가까운 지원 사이즈. 생성 후 **16:9(1920×1080)로 크롭/리사이즈**하여 .pptx 배경에 정합.
- **시드/스타일 고정:** 템플릿 간 일관성을 위해 동일 스타일 프롬프트 + 가능하면 동일 seed 사용 권장.
- **반복 호출 금지 원칙:** 한 템플릿당 1회 생성(비용·일관성). 결과가 어긋나면 도형 묘사를 더 명시적으로 수정 후 1회 재생성.

## (E) codex 실행 지시서 (하이브리드 파이프라인)

> codex에게 주는 최종 작업 지시. 3단계 파이프라인.

### E-0. 환경
- Python + `python-pptx` + OpenAI 이미지 생성 API(gpt-image-2).
- 산출물: `lecture/전반부_AI원론_PPT.pptx`(17장).
- 입력: 본 문서(토큰·템플릿·매핑) + [`전반부_slide_storyline.md`](./전반부_slide_storyline.md)(콘텐츠) + 강사 제공 캡처 PNG(S07/S08/S14).

### E-1. 단계 1 — gpt-image-2로 템플릿 골격 9종 생성
- T1–T9 각각 D절 마스터 프롬프트 + B절 도형 묘사로 **1장씩** 이미지 생성 → `templates/T1.png … T9.png`.
- 16:9로 크롭. 텍스트가 렌더링됐다면 **그 자리는 빈 슬롯으로 다시 생성**(한글/문장이 보이면 재생성).

### E-2. 단계 2 — python-pptx로 17장 조립
- 슬라이드 크기 13.333×7.5in(16:9).
- **각 슬라이드:** (a) 대응 템플릿 이미지를 **전체 배경**으로 `add_picture`(전체 슬라이드 덮개). (b) C절 매핑의 한글 텍스트를 B절 템플릿의 텍스트박스 좌표에 얹음(`add_textbox` + A절 토큰).
- **캡처 슬라이드(S07/S08/S14):** 템플릿의 캡처 프레임 영역(6,22,58,60)에 강사 PNG를 `add_picture`로 맞춤(프레임 안쪽).
- **rich text 강조:** S03 "*말로만*", S09 정규화 박스, S12/S16 하단 굵게 등은 `run.font.color.rgb = ACCENT` / `bold=True`로 강조.

### E-3. 단계 3 — 검증 & 한글 안정화
- F절 한글 타이포 체크리스트 통과.
- 모든 슬라이드에 A-4 반복 컴포넌트(로고·페이지번호·출처) 있는지 확인(표지·명제·진입표는 변형 허용).
- 미러링 가독: 폰트 18pt↑, 캡처 작아지지 않게.

### E-4. python-pptx 합성 핵심 스니펫(골격 — codex가 확장)
```python
from pptx import Presentation
from pptx.util import Inches, Pt
from pptx.dml.color import RGBColor
from pptx.enum.text import PP_ALIGN

# 토큰
INK   = RGBColor(0x1A,0x1A,0x1A)
BODY  = RGBColor(0x4A,0x4A,0x4A)
MUTE  = RGBColor(0x6B,0x72,0x80)
LINE  = RGBColor(0xE5,0xE5,0xE5)
ACC   = RGBColor(0xFF,0x4D,0x30)
SW    = 13.333  # in
SH    = 7.5

def pct(x_pct, dim_in):  # % → inch
    return Inches(x_pct/100*dim_in)

prs = Presentation()
prs.slide_width  = Inches(SW)
prs.slide_height = Inches(SH)
blank = prs.slide_layouts[6]

def add_slide(template_png, boxes):
    s = prs.slides.add_slide(blank)
    s.shapes.add_picture(template_png, 0, 0, Inches(SW), Inches(SH))  # 배경 템플릿
    for b in boxes:
        tx = s.shapes.add_textbox(pct(b['x'],SW), pct(b['y'],SH),
                                  pct(b['w'],SW), pct(b['h'],SH))
        tf = tx.text_frame; tf.word_wrap = True
        p = tf.paragraphs[0]; p.alignment = b.get('align', PP_ALIGN.LEFT)
        r = p.add_run(); r.text = b['text']
        r.font.size = Pt(b['pt']); r.font.bold = b.get('bold',False)
        r.font.color.rgb = b.get('color', BODY)
        r.font.name = 'Pretendard'
    return s
```
> 각 슬라이드의 `boxes` 리스트는 B절 텍스트박스 표 + C절 한글 텍스트로 조립.

## (F) 한글 타이포그래피 규칙 (gpt-image 약점 보완의 핵심)

> 한글은 영문보다 같은 pt에서 **시각 면적이 크고 줄 간격이 더 필요**. 아래 규칙으로 깨짐·잘림·빽빽함 방지.

1. **1줄 권장 한글 자수 공식:** `≈ 박스폭(inch) × 72 ÷ pt × 0.82`. 초과하면 자동 줄바꿈 → 박스 높이(H%)가 부족해 잘림.
   - 예: 폭 8.0in × 22pt → `8×72/22×0.82 ≈ 21자`. 1줄에 21자 이하 권장.
2. **줄간격:** 본문 `1.3~1.4`, 빅스테이트먼트(S03) `1.4~1.5`(`line_spacing`).
3. **pt 보정:** 영문 원본(Nexus) 대비 **한글은 -10~15% 작게** 또는 박스를 더 넓게. 특히 라벨·표 셀.
4. **rich text 강조:** 한 문장 안 일부를 ACCENT/굵게(예: S03 "*말로만*")는 별도 run으로 분할.
5. **영문 혼용:** "CLAUDE.md"·"RAG"·"LLM"·"UX/AX"·"Skill"·숫자·`/세법질의 제7조`는 한글 run과 별도 run(같은 Pretendard 또는 Inter)으로 — 한글 폰트가 영문까지 커버하므로 통일 가능.
6. **자간(letter spacing):** 영문 대문자 라벨(`LABEL`)만 `+2~3pt` 넓게. 한글 본문은 기본.
7. **분리 금지:** 조문번호·날짜(2028.12.31.)·퍼센트는 한 run(줄바꿈으로 쪼개지지 않게).
