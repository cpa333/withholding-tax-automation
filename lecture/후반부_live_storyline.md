# 후반부 Claude Code 라이브 — 실연 runbook (제작 청사진)

> 회계사 대상 AI 원론 강의(v3.0 → **v3.1 난이도 보정**) **후반부 Claude Code 라이브 30분(30:00–60:00)**의 실연 지시서.
> 전반부가 PPT 슬라이드 청사진이었다면, 후반부는 **강사가 실제 Claude Code를 라이브로 조작하는 데모**이므로 —
> **비트별 사전 세팅 · 라이브 액션 · 화면 상태 · 강사 멘트 · 시간 · 사고 대비**로 구성된 runbook.
> 모회: [`ai_fundamentals_lecture_plan.md`](./ai_fundamentals_lecture_plan.md). 병렬: [`전반부_slide_storyline.md`](./전반부_slide_storyline.md)(S01–S17, S17 진입표에서 본 문서로 인계).
>
> **v3.1 난이도 보정(2026-06-19)**: 스파르타 5주 유료 과정(벤치마크)과의 비교 검증 반영. ① 비트③ Skill은 **"실행만"으로 축소**(7.5→4분) — SKILL.md 파일(경로·YAML)을 **화면에 미노출**, "만드는 법"은 유료로 미룸. ② **비트④ 서브에이전트 제거**(멘트 1줄로 흡수). ③ 전환 가교 확대(1.5→2.5분, "웹 브라우저 vs 에디터" 비유). ④ 비트② **폰 따라하기 기본화**("CLAUDE.md 한 줄 라벨" 맛보기 — 능력 체감은 주되 자급자족은 안 됨). ⑤ **실물 아티팩트(CLAUDE.md·SKILL.md) 풀텍스트 강의 후 공유 금지**. ⑥ 클로징에 "설치·환경 세팅도 유료에서" 한 줄 + Q&A 버퍼.
> 작성 기준일: 2026-06-18(v3.1 보정 2026-06-19). 조문·세율은 **2026년 세법개정** 기준 — 강사는 리허설 전 law.go.kr 재확인 필수(정직성 세이프가드).

## (B) 비트별 runbook

> 표기: **사전 세팅** / **라이브 액션**(단계별, 한 줄 명령) / **화면 상태** / **강사 멘트**(격식체) / **시간** / **사고 대비** / **전환·CTA**

### B-0. 전환 가교 — "웹 AI → 내 컴퓨터에서 도구까지 쓰는 AI" (30:00–32:30, 2.5분) · v3.1 확대

- **사전 세팅:** `/회계실데모` 폴더 오픈 준비. 웹 AI 창 닫아둔 채 시작. 캡처 `01_전환.png` 대기.
- **라이브 액션:**
  1. 웹 AI 창 닫기(또는 최소화) — "웹은 여기까지"
  2. VS Code(또는 에디터) 실행 → `/회계실데모` 폴더 열기
  3. **★ "웹 브라우저 vs 에디터" 비유(v3.1 추가):** "웹 ChatGPT는 브라우저에서 쓰는 거죠? Claude Code는 제 컴퓨터의 **에디터라는 창**에서 쓰는 겁니다 — 워드 문서 열 듯이요. 어려운 게 아닙니다."
  4. 에디터 내 통합 터미널(또는 Claude Code 창)에서 **한 줄 입력**: `claude`
  5. 세션 시작 확인 — "이게 Claude Code입니다. 제 컴퓨터에서 직접 돕니다."
- **화면 상태:** 웹 AI(닫기) → 에디터 → `/회계실데모` 폴더 트리(분개장·조문·세법질의 샘플) → `claude` 입력 → 세션 프롬프트
- **강사 멘트:** "웹 AI는 매번 역할·맥락을 다시 설명해야 하고, 내 파일을 직접 만지지 못하고, 어제랑 오늘 결과가 다릅니다. 이걸 넘는 도구가 Claude Code — 제 컴퓨터에서 직접 돌아가는 AI입니다. **웹 브라우저 대신 에디터 창에서 쓰는 거라, 워드 문서 여는 것만큼 익숙한 겁니다.** 제가 미리 회계 자료를 넣어둔 폴더입니다 — ABC상사 더미 분개장, 조문 텍스트, 세법 질의 샘플. 지금부터 직접 써보겠습니다."
- **시간 배분:** 멘트(웹 한계+에디터 비유) 1.5분 + 폴더 열기·`claude` 입력·세션 확인 1분
- **사고 대비:** 와이파이 불안→핫스팟(30초, "잠시 네트워크 바꾸겠습니다"); `claude` 안 열림/로그인 만료→캡처 `01_전환.png`("미리 캡처해 둔 화면입니다 — 세팅이 조금 걸려서요. 흐름은 같습니다").
- **전환:** "먼저 전반부에 캡처로 본 환각을, 이 안에서 라이브로 한 번 재현하겠습니다." → 비트①.

### B-1. 비트① 환각 라이브 → 프롬프트 정정 (32:30–39:30, 7분)

- **사전 세팅:** `조문/법인세법_요약.txt` 열어둠; "말로만"·"4요소+3강화" 질문을 **메모장 두 번째 탭에 미리 써둠**((C)-3); 캡처 `02`·`03` 대기.
- **라이브 액션:**
  1. **"말로만" 질문** 복붙("법인세법 조세액공제 공제율이 어떻게 되나요?")
  2. 그럴듯하게 틀린 답 관찰
  3. 멘트 "전반부 캡처 장면, 지금 직접 돌려봤습니다"
  4. **4요소+3강화 질문**으로 다시 입력(역할=세무사·"근거 조문 정확히, 모르면 모른다고 해, 단계별로 생각해")
  5. 정정 답 관찰
  6. 멘트 "물어보는 법을 바꿨더니 스스로 정정했습니다"
- **화면 상태:** Claude Code 세션(질문→응답). 조문은 필요시 두 번째 패널에 대조. 터미널 노출 최소.
- **강사 멘트:** "전반부에 캡처로 보여드린 환각, Claude Code에서도 똑같이 터집니다 — 말로만 물으면 그럴듯하게 틀립니다. 그런데 역할을 세무사로 정해주고 '근거 조문을 정확히, 모르면 모른다고 해, 단계별로 생각해'라고 다시 쓰니 — 스스로 정정했습니다. 물어보는 법이 결과를 바꿉니다."
- **시간 배분:** 말로만 질문+응답 2 / 관찰·멘트 1 / 4요소+정정 2.5 / 정정 관찰·멘트 1 / CTA 미세 1 0.5
- **사고 대비:** **환각 안 터짐**→"오늘 운 좋게 맞았는데, 재현이 안 되는 게 환각의 무서운 점입니다"→캡처 `02`(정직히 "재현 안 됨" 인정). **정정 안 됨**→"이번엔 정정이 안 됐네요 — 이런 경우도 있습니다, 최종 검산은 회계사님 몫"→캡처 `03`. 응답 지연(10초↑)→"생각하는 시간이 좀 걸리네요" 1회 대기. 네트워크→핫스팟 후 재시도 1회, 안 되면 캡처.
- **전환·CTA:** **CTA 미세 1(갈증 씨앗)**: "근데 이 4요소, **매번 다시 쳐야 하죠?** 어제랑 오늘 결과가 다르고? — 이 답답함을 풀어주는 게 다음 장면입니다." → 비트②.

### B-2. ★ 비트② CLAUDE.md 만들기 + 폰 따라하기 (결정적 한 장면) (39:30–49:30, 10분·중심) · v3.1 폰 따라하기 기본화

- **사전 세팅:**
  - `CLAUDE.md`는 **라이브에서 빈 파일부터 만드는 게 핵심**(반영 전 재현) — 폴더엔 안 두되 임시 보관.
  - 업무지시서 풀텍스트(C-1, **강사 내부용·강의 후 공유 금지**)를 **메모장 두 번째 탭에 미리 써둠**(타이핑 시 시간 초과).
  - **★ 폰 따라하기용 "CLAUDE.md 한 줄 라벨"**(역할·말투만 — QR/링크/인쇄물) 준비(v3.1 기본 진행).
  - 비교용 질문 준비; 캡처 `04`·`05` 대기.
- **라이브 액션:**

```
당신은 중소기업 수감 주력 10년 차 세무사입니다.
질문: 법인세법상 중소기업 특별세액감면(조세특례제한법)의 공제율과 적용시기를 알려주세요.

```

## (D) 사고 대비 의사결정 트리 (failure-mode)

```
라이브 중 이상
├─ 환각 안 터짐 → "오늘 운 좋게 맞았는데, 재현이 안 되는 게 환각의 무서운 점" → 캡처 02 *(정직히 "재현 안 됨" 인정)*
├─ 정정 안 됨 → "이번엔 정정이 안 됐네요 — 이런 경우도 있습니다" → 캡처 03
├─ CLAUDE.md 반영 안 됨 → 파일명·경로 확인 → 세션 재시작 1회 → 안 되면 캡처 05 "기술적 이슈 — 미리 캡처한 겁니다"
├─ Skill 미인식 → .claude/skills/ 경로·SKILL.md 확인(강사만, 화면 비추지 않고) → 세션 재시작 1회 → 안 되면 캡처 06
├─ 폰 따라하기 실패(네트워크/참가자 환경) → "강사 화면으로 같이 보시죠" 즉시 회수 (폰 데이터로도 가능, 리스크 낮음)
├─ 응답 지연(10초↑) → "생각하는 시간이 좀 걸리네요" 1회 대기 → 그 넘기면 캡처
├─ 네트워크 끊김 → 핫스팟 전환(30초) → 재시도 1회 → 안 되면 해당 비트 캡처
├─ API 한도/세션 멈춤 → 해당 비트 캡처. "미리 캡처해 둔 겁니다 — 흐름은 같습니다"
└─ 최악(전부 실패) → 전부 캡처본 진행. "오늘 네트워크가 안 좋아서 미리 캡처해 둔 화면으로 보여드리겠습니다 — 내용은 동일" → 캡처 01~06 순서. 클로징은 PPT로.
```
**캡처 전환 멘트 템플릿(공통):** "제가 미리 캡처해 둔 겁니다 — [재현이 안 돼서 / 네트워크가 불안해서 / 세팅이 조금 걸려서]. 흐름은 같습니다."
**정직성 원칙(★):** 캡처로 전환할 때 "라이브로 되는 것처럼" 연출하지 않는다 — 솔직하게 "미리 캡처한 것"이라고 말한다(회계사는 한계를 아는 사람에게 신뢰를 준다).
