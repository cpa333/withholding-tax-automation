# 후반부 Claude Code 라이브 — 실연 runbook (제작 청사진)

> 회계사 대상 AI 원론 강의(v3.0) **후반부 Claude Code 라이브 30분(30:00–60:00)**의 실연 지시서.
> 전반부가 PPT 슬라이드 청사진이었다면, 후반부는 **강사가 실제 Claude Code를 라이브로 조작하는 데모**이므로 —
> **비트별 사전 세팅 · 라이브 액션 · 화면 상태 · 강사 멘트 · 시간 · 사고 대비**로 구성된 runbook.
> 모회: [`ai_fundamentals_lecture_plan.md`](./ai_fundamentals_lecture_plan.md)(v3.0). 병렬: [`전반부_slide_storyline.md`](./전반부_slide_storyline.md)(S01–S17, S17 진입표에서 본 문서로 인계).
> 작성 기준일: 2026-06-18. 조문·세율은 **2026년 세법개정**(2025.12. 국회 통과, 2026.1.1. 이후 사업연도/지급분 적용) 기준 — 강사는 리허설 전 law.go.kr에서 재확인 필수(정직성 세이프가드).

## (B) 비트별 runbook

> 표기: **사전 세팅** / **라이브 액션**(단계별 넘버링, 강사가 치는 명령은 한 줄) / **화면 상태**(청중에게 보이는 창) / **강사 멘트**(핵심 한 줄, 격식체) / **시간**(세부 배분) / **사고 대비**(실패 모드별 즉시 조치 + 캡처 전환 멘트) / **전환·CTA**(다음 비트 다리 + 해당 CTA 미세)

### B-0. 전환 가교 — "웹 AI → 내 컴퓨터에서 도구까지 쓰는 AI" (30:00–31:30, 1.5분)

- **사전 세팅:** `/회계실데모` 폴더 오픈 준비 완료. 웹 AI(ChatGPT/Claude 웹) 창은 닫아둔 상태에서 시작. 캡처 `01_전환.png` 대기.
- **라이브 액션:**
  1. 웹 AI 창 닫기(또는 최소화) — "웹은 여기까지"
  2. VS Code(또는 에디터) 실행 → `/회계실데모` 폴더 열기
  3. 에디터 내 통합 터미널(또는 Claude Code 창)에서 **한 줄 입력**: `claude`
  4. 세션 시작 확인 — "이게 Claude Code입니다. 제 컴퓨터에서 직접 돕니다."
- **화면 상태:** 처음엔 웹 AI 창(닫는 동작) → 에디터로 전환 → `/회계실데모` 폴더 트리(분개장 CSV·조문 텍스트·세법 질의 샘플) → `claude` 입력 → Claude Code 세션 프롬프트
- **강사 멘트:** "웹 AI는 매번 역할·맥락을 다시 설명해야 하고, 내 컴퓨터의 파일을 직접 만지지 못하고, 어제랑 오늘 결과가 다릅니다. 이걸 넘어서는 도구가 있습니다. Claude Code라고, 제 컴퓨터에서 직접 돌아가는 AI입니다. 제가 미리 회계 자료를 넣어둔 폴더입니다 — ABC상사 더미 분개장, 조문 텍스트, 세법 질의 샘플. 지금부터 30분은 이걸 제가 직접 써보겠습니다."
- **시간 배분:** 멘트 45초 + 폴더 열기·`claude` 입력·세션 확인 45초
- **사고 대비:**
  - 와이파이 불안 → 핫스팟 전환(30초). 멘트: "잠시 네트워크 바꾸겠습니다."
  - `claude` 명령 안 열림/로그인 만료 → 캡처 `01_전환.png`로 전환. 멘트: "제가 미리 캡처해 둔 화면입니다 — 세팅이 조금 걸려서요. 흐름은 같습니다."
- **전환·CTA:** "먼저 전반부에 캡처로 보여드린 환각을, 이 안에서 라이브로 한 번 재현하겠습니다." → 비트① 진입. (이 비트엔 CTA 미세 없음)

### B-1. 비트① 환각 라이브 → 프롬프트 정정 (31:30–38:30, 7분)

- **사전 세팅:**
  - `/회계실데모/조문/법인세법_요약.txt` 열어둔 상태(또는 빠르게 열 수 있게)
  - "말로만" 버전 질문과 "4요소+3강화" 버전 질문을 **메모장/에디터 두 번째 탭**에 미리 써둠(라이브에서 타이핑하면 시간 초과). (C)-3 질문 세트 참조.
  - 캡처 `02_환각틀림.png`·`03_정정답.png` 대기
- **라이브 액션:**
  1. Claude Code 세션에 **"말로만" 버전 질문** 복붙(또는 타이핑) — "법인세법 조세액공제 공제율이 어떻게 되나요?" 수준의 짧은 질문
  2. AI 응답 대기 — 그럴듯하게 틀린 답(공제율 미세 왜곡 또는 적용시기 틀림) 관찰
  3. 멘트: "전반부 캡처 장면, 지금 직접 한 번 돌려봤습니다. 얼핏 맞아 보이시죠?"
  4. **4요소+3강화 버전 질문**으로 다시 입력(역할=세무사 / "근거 조문 정확히 인용, 모르면 모른다고 해, 단계별로 생각해")
  5. AI가 정정하는 답 관찰 — 조문 정확 인용, "모르겠습니다" 또는 보수적 답
  6. 멘트: "물어보는 법을 바꿨더니 스스로 정정했습니다."
- **화면 상태:** Claude Code 세션 창(질문 입력 → 응답 출력). 조문 파일은 필요시 에디터 두 번째 패널에 띄워 "근거" 대조. 터미널 노출 최소.
- **강사 멘트(핵심):** "전반부에 캡처로 보여드린 환각, Claude Code 안에서도 똑같이 터집니다 — 말로만 물으면 그럴듯하게 틀립니다. 그런데 역할을 세무사로 정해주고, '근거 조문을 정확히 인용, 모르면 모른다고 해, 단계별로 생각해'라고 다시 쓰니 — 스스로 정정했습니다. 물어보는 법이 결과를 바꿉니다."
- **시간 배분:** 말로만 질문+응답 2분 / 관찰·멘트 1분 / 4요소 질문+정정 응답 2.5분 / 정정 관찰·멘트 1분 / CTA 미세 1 0.5분
- **사고 대비:**
  - **환각이 안 터짐**(말로만 물어도 정답) → 멘트: "오늘은 운 좋게 맞았는데, 이게 재현이 안 되는 게 환각의 무서운 점입니다 — 제가 미리 캡처해 둔 틀린 사례를 보여드리겠습니다." → 캡처 `02_환각틀림.png` 전환. *(솔직하게 "재현이 안 된다"고 말하는 게 정직성에 유리.)*
  - **정정이 안 됨**(4요소 써도 틀림) → 멘트: "이번엔 정정이 안 됐는데, 이런 경우도 있습니다 — 그래서 최종 검산은 회계사님 몫입니다." → 캡처 `03_정정답.png`로 "이렇게 정정되는 경우도 있다" 보여줌.
  - **응답 지연**(10초 이상) → "생각하는 시간이 좀 걸리네요" 1회, 그 넘기면 캡처로.
