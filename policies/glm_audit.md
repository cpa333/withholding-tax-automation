## 0. 한 줄 결론 (Executive Summary)

| 판단 항목 | 결론 |
|---|---|
| **무조건 전 기능 판매** | **비권장(No-Go)** — 국민연금 EDI의 명시적 자동화 금지 조항(2024-08-01 시행)을 상품에 포함해 유상 판매하면, "약관 위반을 설계적으로 전제한 상품"이라는 프레임을 쓰기 쉽다. |
| **단계적·조건부 판매** | **Conditional Go** — 위하고 중심 Phase 1 → 건강보험·홈택스 Phase 2 → 고용산재·국민연금 Phase 3(서면 회신 확보 후). |
| **"100% 합법 보장 제품" 마케팅** | **불가** — 현 구조로 도달할 수 있는 최대 위치는 **"적정 통제를 갖춘 B2B 업무 보조 도구"**다. |
| **자가사용(내부) vs 유상 판매(외부)** | 질적으로 다른 리스크. 자가사용은 '이용자 약관 위반'이 중심이나, 판매는 ① 약관 위반 유도·조장 ② 표시광고 문제 ③ 고객 계정 제재로 인한 손해배상 ④ 단속 프레임이 추가된다. |

**한 문장 요약:** 공개 약관 기준으로 "즉시 형사처벌" 대상은 아니다. 그러나 **국민연금 EDI 모듈은 서면 허락 없이 상품에 넣으면 안 되며**, 나머지 모듈도 위하고 약관의 '재판매 금지'와 공단 약관의 '정보 외부 제공 금지'를 직접 건드리지 않도록 **"도구 라이선스 판매 + BYO-credentials + 로컬 처리"** 구조와 마케팅 표현을 계약·기술 양면에서 엄격히 통제해야 한다.

### 1.2 기술·운영 전제 (리스크 평가 기준)

- Windows 설치형 `.exe` (PyInstaller, `installer.iss`)

- **사람 로그인(human-in-the-loop)**: 공동인증서/간편인증은 사용자가 직접 완료, 자동화는 로그인 이후부터

- 데이터·인증정보: **사용자 PC 로컬에만 저장**, 당사 서버로 전송 없음

### 3.1 국민연금 EDI (`edi.nps.or.kr`) — **최대 단일 리스크**

**원문 확인 (제12조 제2항 제12호, 2024-08-01 시행본, 2026-07-17 재확인)**

> "사전 허락 없이 자동화된 수단(매크로, 스크래핑 등)을 이용하여 당 사이트가 제공한 서비스에 로그인을 시도 또는 로그인하는 행위 **(인증정보를 이용하여 로그인 후 개인정보를 수집하는 행위를 포함)**"

**판단 표**

| 쟁점 | 결론 | 근거 |
|---|---|---|
| 사람 로그인 후 자동 다운로드 | 문언상 **제12호 괄호에 걸침**. "해당 없음" 단정 불가 | 조문 명시 |
| 사전 허락 | 조문상 **가장 확실한 해소 조건** | "사전 허락 없이"가 요건 |
| stealth / 봇 탐지 회피 (`src/utils/stealth.py`) | 제13호 및 보호조치 우회 프레임과 맞닿음 → **상용판에서 제거 강권** | 보안장치 무력화 금지 |
| **GLM 독자 발견**: NPS "자료수집 허용기간" | 공단이 **전면 금지가 아닌 조건부 허용** 정책을 운영 중 | 위하고 공지 교차확인(위 자료 D) |

**대응**
1. 메인 SKU에서 **국민연금 모듈 제외** 또는 공단 **서면 허락(국민신문고 질의) 후 옵션 활성화**.
2. 허락 회신 전까지는 사용자가 **공단이 공지하는 "자료수집 허용기간" 내에서만** 수동과 동일한 속도·물량으로 운용.
3. `stealth` 기능은 자가사용이든 상용이든 제거 권고 — "보안장치 우회" 프레임 차단.

- 제11장(제54조·제55조): 세무대리인의 수임납세자 세무정보 조회 절차 규정 (별지 제4호 서식 동의서)

## 4. 상용 판매 관점 법리 분석 (B2B seat 구독)

> 자가사용과 **판매는 질적으로 다른 리스크**. 이 장이 본 보고서의 핵심이다.

### 4.1 "도구 판매 vs 서비스 재판매" 구분 원칙

**원칙(유효 가능성 높음)**
- "도구(tool) 제공"과 "도구로 자동화 수행(execution)"의 분리는 저작권·정보통신망법·부정경쟁방지법 판례에서 폭넉게 인정
- 귀하 모델(① 인증정보 사용자 로컬만, ② 데이터 자동화 도구 경유 안 함, ③ 판매 = 도구 라이선스 부여이지 자동화 수행 위탁 아님)은 이 분리 원칙에 부합

**분리가 무너지는 시나리오 (피해야 할 구조)**
| 시나리오 | 법적 의미 |
|---|---|
| "우리가 회계법인 대신 위하고에 대신 입력해 드립니다" (managed service) | 도구 벤더가 아닌 **자동화 수행자**로 약관 직접 구속 |
| 도구가 위하고/공단 화면·API 역설계 후 캡처·재배포 | 복제·배포 금지 조항 직접 위반 |
| 인증서·비밀번호 평문 저장·외부 전송 | 전자서명법 제16조(대여 금지)·개인정보보호법 위반 노출 |
| 서명·제출을 도구가 임의 수행(사용자 개입 없음) | 전자서명법 신뢰성 훼손 |

### 4.2 부정경쟁방지법 제2조 제1호 차목 (타인의 기술적 성과 무단이용)

| 요건 | 귀하 경우 | 평가 |
|---|---|---|
| 타인의 상당한 투자·노력 성과물 | 위하고 서비스, 공단 시스템 | 해당 가능성 |
| **부정한 방법으로 취득·사용** | **도구만 판매, 위하고/공단 데이터 취득·사용 안 함** | **미충족 가능성 높음** |
| 경쟁관계 | 위하고와 직접 경쟁 아님; 공단과 경쟁 불가 | 약함 |
| 타인 경제적 이익 침해 | 위하고 매출 직접 잠식 불명확 | 약함 |

**결론**: 대법 2019도14725(야놀자/여기어때)는 데이터 무단수집·DB화 사안. "도구 판매 자체를 위법으로 본 한국 판례는 확인되지 않음." 다만 도구가 위하고 API를 역분석해 그 성과를 자동화에 활용하고 위하고 RPA 매출을 잠식한다는 주장이 이론적으로 가능하나, 요건 입증 부담이 커서 유죄 입장은 어려움.

### 4.3 공공누리(KOGL) 라이선스 — **기대 불가**

- KOGL은 '공공저작물'의 저작권 자유이용 허락 라이선스
- 홈택스·공단 EDI의 납세자 신고정보·사업장 가입자 현황·신고 이력 등은 **'저작물'이 아니라 개인정보·세무정보(비저작물)**
- 개인정보보호법·세법 비밀유지의무 등 별도 강한 규제 대상
- **결론**: "KOGL Type 1로 출처 표시하면 상업 자동화 가능" 논리는 홈택스/공단 EDI에 기대 어려움.

### 4.4 RPA·브라우저 자동화 상용 도구 선례

- **UiPath, Automation Anywhere, Microsoft Power Automate**: 타겟 애플리케이션을 특정하지 않는 범용 RPA — 특정 SaaS 사전 허가 없이 판매, 분쟁 드물어
- **Selenium/Playwright/Puppeteer**: 오픈소스, 상용 제품(BrowserStack, Apify)도 그 위에 구축
- **한국 인접 사례**: 데이터 무단 수집·DB화 사건(야놀자/여기어때, 티몬/위메프)이 대부분. **"도구 자체 상업 판매를 처벌한 한국 판례는 확인 안 됨"**
- 시사점: **'중립적 범용 도구' 포지셔닝**이 분쟁 회피의 실무적 관행. 반대로 '특정 타인 서비스 자동화/스크래핑'을 핵심 가치로 마케팅하면 표적.
