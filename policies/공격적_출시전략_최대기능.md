## 1. 핵심 전략 요약 — 무엇을 더 살리고(공격), 무엇은 진짜 못 하는가(정직)

보수합본의 결론은 한마디로 **"위험한 건 빼고, EDI는 허락받은 뒤에, 단계적으로 뒤 Stage에서 팔아라"**였다. 본 공격안의 결론은 다르다:

> **비-EDI 3개 사이트(WEHAGO·홈택스·COMWEL)를 "축소판"이 아니라 "풀기능 상용 SKU"로, 단 하나의 자가검증 게이트만 통과하면 이번 분기에 출시한다. EDI 2개(NHIS·NPS)와 완전 무인 제출은 — 정직하게 — 보수합본과 똑같이 지금은 못 살린다.**

공격의 순수익은 "금지·차단 사이트를 허락 없이 켜는 것"이 **아니라**, "차단도 금지도 없는 3개 사이트의 기능 표면을 **축소판에서 파괴적 동작 포함 풀기능으로 넓히고**, 출시 시점을 **뒤 Stage에서 이번 분기로 앞당기는 것**"이다. 이 둘을 혼동하면 위법이 된다.

### 이 전략을 관통하는 핵심 장치: 3개의 게이트

레드팀이 모든 클러스터에서 공통으로 잡아낸 오류는 **"지금 출시 가능"이 현재 코드 상태가 아니라 재설계 완료 + 라이브 검증 통과 후의 미래형**이라는 점이다. 이를 3개 게이트로 정리하면 전략 전체가 정합적으로 맞물린다.

| 게이트 | 내용 | 통제 주체 | 소요 | 이걸 여는 대상 |
|---|---|---|---|---|
| **게이트 1** (자가검증형) | 전역 스텔스(playwright-stealth 10모듈 + `--disable-blink-features=AutomationControlled`)를 바이너리에서 **완전 제거** → 스텔스 OFF 상태로 WEHAGO·홈택스·COMWEL 전 워크플로 라이브 회귀 실행 → **무한리로드/로그인루프 부재를 실측·문서화** | **판매자** (통제 가능) | 수일~수주 | WEHAGO·홈택스 (이번 분기) |

## 3. 지금 당장 더 살릴 수 있는 것 — 재설계로 이번 분기 출시

보수합본이 "빼라/미뤄라"고 한 것 중, 재설계로 **이번 분기에 낼 수 있는 것**과 그 구체적 재설계다. **게이트 무관(즉시)** 항목과 **게이트1 통과 후(이번 분기)** 항목을 구분한다.

### 3-0. 게이트 무관 — 지금 즉시 (P0 위생 + 로컬처리)

**(A) 판매 채널 자체를 여는 위생·계약 — 이것 없이는 유료 출시 기능 0개**

- **PyMuPDF(AGPL) 완전 제거** — `pdf_reader.py`(fitz)는 제품코드 어디서도 import되지 않는 **dead code**이고(본 세션 재확인: `pdf_reader.py`를 import하는 제품코드 0건, 실사용은 `raw_data_reader.py:61,66` pdfplumber(BSD)), 카피레프트 배포의무의 유일한 원천이다. 단 `requirements.txt` 한 줄이 아니라 **`build.py:136`(`--collect-submodules=PyMuPDF`), `:137`(`--hidden-import=fitz`), `:247`(PYZ fitz 검증), `:268-269`(`_mupdf.pyd` verify_bundle)를 함께 제거**해야 빌드가 깨지지 않는다. 제거로 카피레프트 배포의무 자체가 소멸.
- **실 Publisher + 코드서명** — `installer.iss:14`의 플레이스홀더 Publisher·`example.com` URL·Defender 예외 요구를 실제 서명으로 대체(공급망·신뢰 문제 해소).
- **시트/조직 라이선스 실구현 + EULA·개인정보처리방침·DPA·전자상거래법 고지·PySide6(LGPLv3) THIRD_PARTY_NOTICES** — 구독 판매 자체를 적법화.
- **EULA 책임배분(약관규제법 한계 내)** — 구매대상을 자격·수임·업무대행 지정 확인된 세무대리인으로 한정, 이용자에게 각 포털 약관준수·최종검토·최종제출 책임 진술보증. **단 판매자의 고의·중과실·설계결함(스텔스·무인제출) 면책은 조각**(약관규제법 제7조, 과도 면책 무효).

**(B) NHIS 로컬 문서처리 SKU — 자동화가 아니므로 게이트 무관**

## 4. 조건 충족 시 살릴 수 있는 것 — 게이트3(각 법인 서면허락)이 여는 기능

여기가 보수합본과 "무엇을 더 살리나"의 경계다. **재설계로 표면이 넓어지지는 않지만**, 각 회계법인 명의 사전허락 온보딩·공식채널 전환으로 **"영구 제외"를 "허락받은 법인 한정 조건부 활성"으로 전환**할 수 있다.

### 4-1. NHIS 자동화 본체 — 게이트3 (능동차단이라 문턱 최고)

- **조건:** ① NHIS가 각 법인이 자기 명의로 "자동화/차단 우회를 명시 인지·승인"하는 서면허락을 발급하는 **공식 채널이 실재함을 공단 서면회신으로 선실증** → ② 특정 테넌트가 그 허락문서를 실제 보유함이 **아웃오브밴드로 검증** → ③ 그 테넌트 **전용 빌드에만** 자동화 모듈 포함.
- ⚠️ **레드팀 교정 반영:** 자동화 모듈(Nexacro 직접호출·좌표/타이밍 지터·webdriver 우회)을 상용 빌드에 **동봉한 채 UI 플래그로 언락하는 구조는 채택 안 함.** flag는 코드를 제거하지 않으므로 "판매 준비된 무력화 도구"(2024도14169 계열 [신뢰도 MEDIUM]) 리스크가 상존한다. **바이너리 물리 제외 + 서버측 게이트 배포(server-side gated delivery)**로만.
- **합법 전환 논리:** 제17조①10호(및 정보통신망법 제48조)는 "공단의 사전 허락 없이"·"운영자 의사에 반하는"을 구성요건으로 두므로, 운영자가 서면 승인하면 그 전제가 깨진다. 허락 귀속을 **판매자 일괄이 아니라 각 법인**으로 두는 것이 핵심.
- **온보딩 페이퍼워크는 지금 내장 가능:** "서면질의 템플릿 생성 + 공단 회신 기록·보관"은 어떤 자동접근도 발생시키지 않으므로 지금 넣어도 된다. **단 이 기능이 자동화 모듈을 언락해서는 안 된다**(레드팀 조건).

### 4-2. NPS 자동화 본체 — 게이트3

- **유일 활성경로(옵션A):** 기본 잠금 출하 + 테넌트별 "공단 사전허락 신청 마법사"(수임 위임장·업무대행 지정 증빙·자동화 방식 설명서 첨부) → 회신문서 업로드 → **서버측 사람검토 → 해당 테넌트만 서버측 언락.** ⚠️ 자기신고 클릭스루 서약으로 언락하는 구조는 채택 안 함(레드팀).
- **활성 전 하드닝 병행:** 업무대행기관 한정 판매(옵션C, 헌재/정당권리자 지위) + 스텔스 제거·사람개시 재구성(옵션D) + 중립 포지셔닝(옵션E). **단 옵션B(자료수집 허용기간 연동)는 legalization 경로가 아니라, 허락 확보 이후에 얹는 제10조10호 부가 하드닝으로만** 사용.
- **합법 전환 논리:** 제12조②12호가 "사전 허락 없이"를 스스로 구성요건으로 두므로 허락받으면 조항 부적용. NPS는 NHIS와 달리 능동 기술차단이 코드/약관상 미확인이라 "허가제" 사이트이지 하드케이스가 아니다(스텔스 OFF로 정상동작 여부는 실측 필요).

### 4-3. COMWEL 화면 자동수집 — 게이트1+2

- **조건:** ① 게이트2(약관 실물 + 서면회신)로 NPS식 명시금지 부재 확인 **AND** ② 게이트1(스텔스 OFF 라이브)로 능동 webdriver 감지 부재 실측. **둘 중 하나라도 실패(약관 금지 존재 or 능동차단 존재)하면 화면 자동수집은 엔지니어링 합법화 불가 → QR 재개 대기 또는 기능 폐기.**
- **자격 gating 필수(헌재 2020헌마139):** 보험사무대행 지정 없는 개인 CPA는 위탁조회 합법경로 자체가 없으므로 **지정번호 원본대조**로 대상 배제(자기신고 불가). 수임처별 사업주 위탁동의서 등록을 온보딩 필수화(개인정보보호법 제26조).
- **QR 공식채널은 로드맵으로 강등** — ⚠️ 레드팀 교정: 2025-09 개편으로 현재 중단, 커버범위가 "월별보험료 산출내역서"뿐(제품이 뽑는 PA20209 부과내역·지원금 미커버), 제3자 제품 직접접근 인터페이스 미확인. "1순위·안전"으로 세우지 않고, 공단 서면회신으로 재개·커버확대·접근방식 확인 후에만 대체경로로 승격.

## 부록 A. 레드팀이 걸러 채택하지 않은 제안 (의사결정자용)

"왜 이건 더 공격적으로 안 하냐"는 질문에 대한 정직한 답. 아래는 **매력적이지만 선을 넘어 제외**했다.

1. **NPS "자료수집 허용기간 연동"만으로 단독 출시** — 허용기간 준수는 제10조10호(정상운영 지장) 방어일 뿐 제12조②12호를 해소 못 함. 허락 후 하드닝 레이어로만.
2. **자기신고 클릭스루 서약으로 EDI 언락** — 계약 면책은 실제 위반을 치유 못 하고 과도 시 약관규제법 제6·7조로 무효. 서버측 사람검토 언락으로 대체.
3. **홈택스 예외기반 clean-건 자동제출 기본값** — 다수 정상신고를 수치 무노출로 1클릭 제출 = 형식적 게이트. 옵션4 단일 기본경로로 대체.
4. **비차단 사이트에 webdriver 억제 유지(CDP 호환성 재라벨링)** — 얻는 기능 0 + 은닉코드 상존이 고의 정황증거화(EV 음수). 제거가 유리.
5. **WEHAGO 파괴적 동작 배치 1클릭 승인** — 대상별 개별승인으로 되돌림.
6. **React 내부상태 주입 유지한 채 "순수 사람조작 재현" 마케팅** — 내부주입이 잔존하면 "정상 UI 사용" 방어·마케팅 불일치. 가시 DOM 조작으로 실제 교체 후에만 주장 가능.
7. **EDI 모듈 바이너리 동봉 + flag-off** — 우회코드 물리 존재 = "판매 준비된 무력화 도구". 바이너리 제외 + 서버측 게이트 배포.
8. **COMWEL QR을 1순위·"안전"으로 채택** — 현재 중단 + 부분 커버. 로드맵으로 강등.
